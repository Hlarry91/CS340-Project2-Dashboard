# CS 340 Project Two – Dashboard

## Reflection

### Maintainable, Readable, Adaptable Code
I write programs that are maintainable and readable by organizing code into clear, reusable components and using consistent naming conventions. In this project, the CRUD Python module allowed me to separate database logic from the dashboard interface. This made the code easier to manage and update without affecting other parts of the application. The advantage of this approach is that the same module can be reused in future projects that require database connections, reducing development time and improving consistency.

### Approach as a Computer Scientist
When approaching this project, I focused on understanding the client requirements first and then breaking the problem into smaller parts, such as database setup, query development, and dashboard design. Compared to previous courses, this project required more real-world thinking and integration between systems. I approached this differently by thinking from the user and client perspective, ensuring the dashboard was functional and easy to use. In the future, I would continue using structured planning, testing queries early, and designing flexible data models to meet different client needs.

### Role of Computer Scientists
Computer scientists design and build systems that help organizations manage and analyze data efficiently. This work is important because it allows companies to make informed decisions and improve operations. In this project, the dashboard helps Grazioso Salvare quickly identify suitable dogs for rescue training, which improves their ability to respond to real-world situations. This type of solution increases efficiency, accuracy, and overall effectiveness.
